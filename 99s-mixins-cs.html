Mixins in C#

Realisierung durch Erweiterungsmethoden (_extension methods_). ```csharp class Message { public string Text { get; set; } } `static class` Mixins { `public static` string Escalated(`this Message self`) { return self.Text.ToUpper(); } `public static` void Concat(`this Message self`, `Message other`) `=>` self.Text = self.Text + " " + other.Text; static void Main(string[] args) { Message m = new Message(); m.Text = "Hello, world"; Console.WriteLine(m.Escalated()); // "HELLO, WORLD" m.Concat(m); Console.WriteLine(m.Text); // "Hello, worldHello, world" } } ``` - Öffentliche statische Methode in statischer Klasse - Erstes Argument vom Zieltyp mit Modifier `this` - `=>` für [Einzelanweisung](https://docs.microsoft.com/en-us/dotnet/csharp/language-reference/operators/lambda-operator) (kürzer) --- # Zustandsbehaftete Mixins Bisher sind die Erweiterungsmethoden lediglich statische Methoden ohne separaten Zustand. Verwende `ConditionalWeakTable` als Zustandsspeicher. ```csharp class Message { public string Text { get; set; } } static class MessageMixins { private static readonly ConditionalWeakTable state_ = new ConditionalWeakTable(); public static string Escalate(this Message self) { object n; if (!state_.TryGetValue(self, out n)) n = 0; string message = self.Text.ToUpper() + new String('!', (int) n); state_.AddOrUpdate(self, ((int) n)+1); return message; } } ``` --- # Vorsicht mit Autom. Typumwandlung (1) ```csharp class Vorsicht { public void Obacht(object o) => Console.WriteLine("Vorsicht.Obacht: {0}", o.GetType()); } static class VorsichtMixin { public static void Obacht(this Vorsicht self, int i) => Console.writeLine("VorsichtMixin.Obacht: {0}", i); static void Main(string[] args) { Vorsicht v = new Vorsicht(); v.Obacht(3); // "Vorsicht.Obacht: System.Int32" v.Obacht("Hans"); // "Vorsicht.Obacht: System.String" } } ``` **Vorsicht:** Der Typ `object` trifft hier auf alle Argumente zu! --- # Vorsicht mit Autom. Typumwandlung (2) ```csharp class Vorsicht { public void Obacht(`int i`) => Console.WriteLine("Vorsicht.Obacht: {0}", i); } static class VorsichtMixin { public static void Obacht(this Vorsicht self, `object o`) => Console.writeLine("VorsichtMixin.Obacht: {0}", o.GetType()); static void Main(string[] args) { Vorsicht v = new Vorsicht(); v.Obacht(3); // `Vorsicht.Obacht: System.Int32` v.Obacht("Hans"); // `VorsichtMixin.Obacht: System.String` } } ``` --- # Syntaxzucker Erweiterungsmethoden sind normale statische Methoden _mit syntaktischem Zucker_. Der Compiler erkennt hier über das Schlüsselwort `this` als Argumentenmodifier eine Erweiterung für alle `Message` (und Unterklassen). ```csharp class Message { public string Text { get; } } static class MessageMixin { public static string Escalated(this Message self) => self.Text.ToUpper(); static void Main(string[] args) { Message m = new Message(); // äquivalent: m.Escalated(); MessageMixin.Escalated(m); } } ``` --- # Sichtbarkeiten Da Erweiterungsmethoden nur syntaktischer Zucker sind, gelten die Sichtbarkeiten wie gehabt: ```csharp class Message { public string Text { get; set; } private int id; } static class MessageMixins { public static void Debug(this Message self) { self.Text; // OK self.id; // Compilerfehler: not visible! } } ``` **Daher:** Es ist sinnvoll Erweiterungsmethoden gegen Interfaces zu implementieren, damit - alle benötigten Attribute eingesehen werden können - das Mixin flexibel über das Interface "angeheftet" werden kann --- # Binden und Überschreiben (1) Erweiterungsmethoden _binden wenn nötig_ aber _überschreiben **nicht**_. ```csharp class Message { public string Text { get; set; } public string Escalated() => Text.ToLower(); } static class MessageMixin { public static string Escalated(this Message self) => self.Text.ToUpper(); static void Main(string[] args) { Message m = new Message(); m.Text = "Hello, world"; Console.WriteLine(m.Escalated()); // "hello, world" Console.WriteLine(MessageMixin.Escalated(m)); // "HELLO, WORLD" } } ``` --- # Binden und Überschreiben (2) Erweiterungsmethoden _binden wenn nötig_ aber _überschreiben **nicht**_. ```csharp interface IMessage { string Text { get; } } class Message `: IMessage` { string Text { get; set; } string Escalated() => Text.ToLower(); } static class MessageMixin { public static string Escalated(this `IMessage` self) => self.Text.ToUpper(); static void Main(string[] args) { Message m = new Message(); m.Text = "Hello, world"; Console.WriteLine(m.Escalated()); // "hello, world" Console.WriteLine(`(m as IMessage)`.Escalated()); // "HELLO, WORLD" } } ``` --- # Zusammenfassung - _"Favor composition over inheritance:"_ Bevorzuge Komposition der Vererbung - _Mixins_ erweitern _bestehende_ Klassen _ohne_ eine verwandschaftliche Beziehung herzustellen - Für C# gilt: - Mixins können zu einem gewissen Grad mit _extension methods_ realisiert werden. - Die Einbindung einer `ConditionalWeakTable` erlaubt es instanzspezifische Zustandsinformation zu unterhalten. - Extension methods binden, aber überschreiben nicht; es gelten die spezifizierten Sichtbarkeiten - Vergleiche auch: Traits in Scala, Mehrfachvererbung in C++ oder Defaultmethoden in Java. --- # Weiterführende Informationen #### Dokumentation - #### Literatur - Craig, Iain (2007).
